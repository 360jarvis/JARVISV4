# Jarvixx 2.0 Build Lock

## Current Build Name

Jarvixx 2.0 Clean Foundation

## Core Product Rules

1. This is a multi-tenant SaaS platform.
2. Every record must belong to a tenant.
3. Legacy routing conflicts are not allowed.
4. Buttons must be functional or clearly marked Coming Soon.
5. Approved modules must be locked after owner approval.
6. Major actions must be audit-ready.
7. UI must stay luxury, clean, compact, and professional.
8. The global sidebar must stay consistent across the app.
9. Public website and internal app must remain separate.

- Dates: MM/DD/YYYY, include day of week where useful.
- Phones: (XXX) XXX-XXXX.
- Money: USD currency formatting.
- Tables: compact professional rows, 25 rows per page default.
